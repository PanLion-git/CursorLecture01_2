Команди Git

(термінал)

git config --global user.name blabla

- встановлює ім'я користувача, що відображатиметься у інформацї про виконані зміни

git config --global user.email blabal@bla

- встановлює email користувача, що відображатиметься у інформацї про виконані зміни

git config --global core.editor "''"

- задає редактор для формування повідомлень

git config --list

- виводить перелік налаштувань поточного репозиторія

git init

- створення репозиторію поточної директорії (директорія вказується командою cd "path")

git status

- виводить перелік індексованих файлів, що були модифіковані

git add [. | file_name | regexp]

- додає файли у перелік який буде комітитись

git commit [-a] -m 'commit_info'

- комітить зміни, що були включені командою git add

git log

- виводить перелік комітів

git checkout [branch_name]

- переключається на вказану гілку

git branch [branch_name]

- створення нової гілки (без параметра видає перелік всіх гілок)

git push

- пеносить зміни з локального репозиторію на віддалений

git pull

- переносить зміни з віддаленого репозиторія на локальний

git merge [branch_name]

- заливає зміни з поточної гілки на вказану

git clone

- копіює віддалений репозиторій на локальний комп'ютер

.gitignore - файл, у якому міститься інформація про файли які не повинні потрапляти у віддалений репозиторій. Цей файл створюється за допомогою відповідної опції при створенні віддаленого репозиторія, або у кориневому каталозі локального репозиторія. Таку назву файлу можна отримати шляхом переіменування файлу з ім'ям gitignore.txt у командному вікні, що відкивається по команді cmd за допомогою команд cd 'path_to_local_repository' та rename gitignore.txt .gitignore .
Більш детальну інформацію дивись тут.

git revert [some_option]

- відміняє коміт, [some_option]

git reset

- відміняє проіндексовані зміни (виключає файли, які підготовані для команди git commit)

git rebase [branch_name]

- переносить зміни з поточної гілки на вказану

git cherry-pick [ID-commit]

- пеносить вказаний коміт на поточну гілку

git stash

- приховує зроблені зміни. Відміна останнього приховування командою git stash apply. Відміна заданого приховування виконується по команді git stash apply stash@{2}. Переглянути всі приховування можна по команді git stash list
